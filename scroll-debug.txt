Screenshot shows SEO Automation page on mobile (domainslayer.ai)
- Page shows Strategy dropdown, Timeline buttons, Aggressiveness slider, WordPress Connection, Auto Backlink toggle
- User reports cannot scroll at all - page is frozen/locked

Potential causes:
1. Line 61: `overflow-hidden` on the main content wrapper div - this could block scrolling!
   `<div className="flex-1 flex flex-col overflow-hidden min-w-0">`
   On mobile, this `overflow-hidden` prevents the content from scrolling.
   
2. Line 34: `min-h-screen min-h-[100dvh]` on root - this is fine

3. Line 65: `overflow-y-auto` on main - this should allow scrolling BUT the parent has `overflow-hidden`
   On mobile browsers, the combination of parent `overflow-hidden` and child `overflow-y-auto` can cause issues
   especially when there are touch event conflicts.

4. The page content might be taller than the viewport but the scroll container isn't properly set up for mobile touch scrolling.

FIX: 
- Remove `overflow-hidden` from the main content wrapper
- Add `-webkit-overflow-scrolling: touch` for iOS Safari
- Ensure the scroll container is the main element, not nested
